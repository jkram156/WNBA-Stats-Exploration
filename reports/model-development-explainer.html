The Short Version

The current model asks a simple question: over each team's recent regular-season games, which team has looked stronger by scoring margin, points scored, points allowed, and win rate? It then uses logistic regression to turn those recent-strength signals into a win probability.

The current regular-season baseline is raw_n20_strength_core. It uses the prior 20 games for each team when available. It is intentionally simpler than some of the candidates we tested because the simpler strength-core model held up better after cleaning the game universe.

How We Got Here

1. Built the local data foundation.

We created a resyncable SQLite database from the local wehoop-wnba-raw mirror. The extracted tables that matter most for modeling are games, game_competitors, game_team_box_stats, and teams.

2. Created a leakage-safe rolling evaluator.

Each completed game becomes two team-game rows, one for each team. A row's features are built only from games played before that game. That keeps the backtest honest because the model never sees the result of the game it is trying to predict.

3. Tested rolling windows.

We compared 3, 5, 8, 10, 12, 14, 16, and 20-game windows. Very short windows were too noisy. The useful range clustered around 10 to 20 games, with 20 games becoming the best simple regular-season baseline.

4. Added context and richer box-score families.

We tested home/rest context, shooting, rebounding, turnovers, assists/steals/blocks, matchup edges, and opponent-adjusted form. Some of these helped in slices, but the broader feature sets did not clearly beat the simpler strength core after the game universe was cleaned.

5. Cleaned the game universe.

The evaluator now treats regular-season prediction as regular-season WNBA franchise games only. We exclude postseason games and special-event/non-franchise team ids such as USA, WNBA All-Stars, and other exhibition-style rows.

6. Settled on the current baseline.

After the cleanup, raw_n20_strength_core moved back into the practical lead: log loss 0.6232, Brier 0.2167, accuracy 0.6619, and AUC 0.7115 on the canonical regular-season evaluation.

What The Current Model Uses

The prediction script uses the following current feature set. For an upcoming game, it builds the home team's recent averages, the away team's recent averages, and the home-minus-away difference.

Raw Stat Meaning Features Created Why It Matters
scoreFor Average points scored in the recent window. home_scoreFor_avg, away_scoreFor_avg, scoreFor_diff Captures recent offensive production.
scoreAgainst Average points allowed in the recent window. home_scoreAgainst_avg, away_scoreAgainst_avg, scoreAgainst_diff Captures recent defensive performance, indirectly.
margin Average scoring margin, calculated as points for minus points against. home_margin_avg, away_margin_avg, margin_diff The strongest and most stable signal. Teams that win by more tend to be stronger than teams that only narrowly win or often lose.
winPct Recent winning percentage in the rolling window. home_winPct_avg, away_winPct_avg, winPct_diff Captures recent results directly, though it can be less informative than margin because it ignores how close games were.
In total, the current prediction model uses 12 numeric inputs: 4 stats x 3 versions of each stat. The three versions are home team average, away team average, and the difference.

What We Tested But Do Not Currently Use

Context Features

These were useful in some contexts, especially postseason experiments, but did not beat the current regular-season baseline globally.

Box-Score Families

Shooting and creation/defense had some standalone signal. Once margin and points were already present, they did not improve the main baseline enough to keep.

Rebounding was especially suspicious in the adjusted no-context model, where dropping it improved results. Turnovers were not clearly additive for the current baseline.

This was competitive and sometimes better in confident-pick slices, but after the regular-season cleanup it no longer beat raw n=20 strength core overall.

Evidence That Shaped The Current Choice

Experiment Key Result Decision
Short vs long windows n=3 and n=5 were weaker; n=10 to n=20 was the useful range. Use n=20 for the current simple baseline.
Mixed-window model Mixed windows were close but did not clearly beat the best global window on completed seasons. Keep as an idea, but do not use as the current default.
Grouped ablation Margin/strength carried the clearest signal; removing it hurt log loss. Center the model around margin, points for/against, and win rate.
Regular-season cleanup After removing postseason and special-event rows, raw n=20 strength core led at 0.6232 log loss. Make it the canonical regular-season baseline.
Bootstrap comparison The adjusted candidate was only 0.0006 log-loss points behind/away from raw n=20, with a wide interval. Do not overclaim; the current lead is practical, not mathematically final.
Low-confidence splits 45-55% predictions were noisy and close to random. Treat near-50% picks as toss-ups.

How A Prediction Is Made

Step 1: Find the scheduled game.

The upcoming schedule comes from ESPN schedule tables in SQLite.

Step 2: Build each team's recent history.

For each team, the script looks at games before tipoff only, then takes the most recent 20 regular-season franchise games when available.

Step 3: Calculate the 12 strength inputs.

The script computes recent points for, points against, margin, and win percentage for both teams, plus the home-minus-away difference for each.

Step 4: Train the logistic model.

For each scheduled game, the model is trained on prior regular-season team-game rows. The target is whether the row's team won.

Step 5: Convert strength into probability.

The model outputs the home team's win probability. The away team's probability is treated as the complement. The favorite is whichever side is above 50%.

What The Model Does Not Know Yet

The current model does not include injuries, player availability, starting lineups, minute restrictions, travel distance, betting market expectations, roster continuity, coaching changes, or player-level strength. Those are likely the next meaningful feature frontier.

It also does not currently use home-court advantage directly, even though home court showed signal in earlier full-feature experiments. That is a deliberate simplicity choice for the current baseline after the broader context variants failed to beat raw n=20 strength core overall.

Glossary: Stats And Jargon

These are the terms used in the model reports and what they mean in this project.

Term Plain-English Definition How To Read It Here
Log loss A scoring rule for probability predictions. It rewards confident correct predictions and punishes confident wrong predictions. Lower is better. Random 50/50 is about 0.6931. Our current regular-season baseline is 0.6232, so it is finding real signal.
Brier score The average squared distance between the predicted probability and the actual result. Lower is better. It is another way to judge probability quality. The current baseline was 0.2167.
Accuracy The share of rows where the model picked the winning side when using 50% as the cutoff. Higher is better, but it ignores probability quality. A 51% pick and an 80% pick count the same for accuracy.
AUC Area under the ROC curve. It measures how well the model ranks winners above losers across probability thresholds. 0.50 is random ranking; 1.00 is perfect ranking. The current baseline was around 0.7115.
Calibration Whether predicted probabilities match reality. If the model says 60%, those teams should win about 60% of the time. A model can be accurate at picking winners but still overconfident or underconfident. Calibration checks help catch that.
ECE Expected calibration error. It summarizes how far prediction buckets are from actual win rates. Lower is better. It is useful for checking whether the probabilities themselves are believable.
Rolling window The recent set of games used to describe a team entering a matchup. n=20 means the model looks at up to the previous 20 regular-season franchise games for each team.
Team-game row One game represented from one team's point of view. Every WNBA game creates two rows: one for Team A and one for Team B. The target is whether that row's team won.
Leakage Accidentally letting future information into a prediction. We avoid leakage by using only games played before the game being predicted.
Rolling-origin backtest A historical test that trains on earlier seasons and predicts a later season, then rolls forward. This better simulates real future prediction than randomly mixing old and future games together.
Regularized logistic regression A model that turns numeric inputs into a win probability, with a penalty that keeps weights from getting too extreme. It is simple, interpretable, and a good baseline before trying more complex model classes.
scoreFor Points scored by the team. In the current model, this is averaged over the recent window for each team.
scoreAgainst Points allowed by the team. Lower is usually better. It is the current model's simple defensive signal.
margin Points scored minus points allowed. Positive means the team has recently outscored opponents. This has been one of the strongest signals.
winPct The team's winning percentage inside the rolling window. It captures recent results, but margin often carries richer information because it includes how decisive the games were.
Diff feature A home-minus-away difference for one stat. Example: margin_diff compares the home team's recent margin against the away team's recent margin.

Current Interpretation Rules

Probability Range Label How To Read It
50% to 55% Coin flip The model is barely leaning. Treat this as weak evidence.
55% to 65% Lean The recent-strength profile favors one side, but the edge is modest.
65% and above Solid The model sees a clearer strength gap. Still not a guarantee, just a stronger probability signal.
