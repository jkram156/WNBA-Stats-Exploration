Scope

The current evaluator builds one row per team-game. Features are computed from each team and opponent's previous games only, so the current game's result is never included in the rolling averages.

  • Database: wnba_raw.sqlite
  • Rows loaded: 11,536 team-game rows
  • Current default target seasons: 2023 onward
  • Completed-season check: 2023, 2024, and 2025
  • Partial-season check: includes 2026 data available through the local database
  • Latest context-feature check used --iterations 600 --rolling-through 2025 --skip-mixed

Methods Explored

Global Window Models

One rolling window length is applied to every rolling stat. Candidate windows tested so far: 3, 5, 8, 10, 12, 14, 16, 20.

Mixed Window Model

For each target season, each stat gets its own selected window. The selector uses one-stat validation models on the immediately prior season, then the final model trains on all seasons before the target season.

Model

The baseline predictive model is regularized logistic regression with standardized numeric features. Metrics reported are log loss, Brier score, accuracy, and ROC AUC.

Context Features Added

The evaluator now adds basketball context beyond box-score rolling averages: rest advantage, back-to-back flags, recent win rate, recent home-game rate, recent close-game rate, recent average rest, and point matchup edges.

Calculation Details

The prediction unit is a team-game. A single WNBA game contributes two modeling rows: one row for each team. The target is whether that row's team won the game.

Source Tables

  • games: game date, season, completion status.
  • game_competitors: team, opponent, home/away, score, and winner flag.
  • game_team_box_stats: team-level box-score stats stored in long format.

Stats Used

  • Score for, score against, and scoring margin.
  • Assists, blocks, steals, turnovers.
  • Offensive, defensive, and total rebounds.
  • Field goal percentage, three-point percentage, and free throw percentage.

Row Construction

For team T playing opponent O in game g, the model row uses only games played before g. If either team has fewer than the minimum prior-game threshold, the row is skipped. The current threshold is 3 prior games.

target y(T,g) = 1 if team T won game g 0 otherwise

Rolling Average Formula

For a stat s and window size n, the rolling average is the mean of that team's most recent n games before the current game. The current game is excluded.

roll_avg(T, s, g, n) = mean(value(T, s, h)) for h in the last n games played by T before game g

For every rolling stat, the feature vector includes the team's rolling value, the opponent's rolling value, and the difference between them.

team_value = roll_avg(T, s, g, n) opponent_value = roll_avg(O, s, g, n) diff_value = team_value - opponent_value

Feature Vector

Each row starts with context features, then appends rolling stat features. Rest days are capped at 14 days so long offseason gaps do not dominate the scale.

x(T,g) = [ home_indicator(T,g), rest_days(T,g), rest_days(O,g), rest_days(T,g) - rest_days(O,g), team_back_to_back, opponent_back_to_back, team_scoreFor_avg, opponent_scoreFor_avg, scoreFor_diff, team_scoreAgainst_avg, opponent_scoreAgainst_avg, scoreAgainst_diff, recent_win_pct_diff, recent_home_rate_diff, recent_close_game_rate_diff, team_scoreFor_avg - opponent_scoreAgainst_avg, ... team_turnovers_avg, opponent_turnovers_avg, turnovers_diff ]

Standardization

Features are standardized using the training split only. This prevents information from the target season from influencing the feature scaling.

z_j = (x_j - mean_train(x_j)) / std_train(x_j)

Logistic Regression

The model estimates the probability that the team wins the game. It uses an intercept plus one weight per standardized feature.

linear_score = b + w_1 z_1 + w_2 z_2 + ... + w_k z_k predicted_win_probability = 1 / (1 + exp(-linear_score))

The fitted model uses L2 regularization, which penalizes large weights and helps reduce overfitting.

training_objective = average_log_loss + lambda * sum(w_j^2) default lambda = 0.01

Evaluation Metrics

Log loss and Brier score evaluate probability quality, not just correct picks. Lower is better for both. Accuracy uses a simple 0.50 probability cutoff. ROC AUC measures ranking quality; higher is better.

log_loss = -mean(y * log(p) + (1 - y) * log(1 - p)) brier_score = mean((p - y)^2) accuracy = mean(1 if (p >= 0.50) equals y else 0)

Rolling-Origin Backtest

The 2023 onward evaluation is walk-forward by season. For each target season, all earlier seasons are used for training and the target season is used for evaluation.

for target_season in [2023, 2024, 2025]: train = seasons before target_season test = target_season

Mixed-Window Selection

The mixed model chooses a different window for each stat without looking at the target season. For a target season, each stat's window is chosen by testing one-stat models on the immediately prior season. After those windows are selected, the final mixed model trains on all earlier seasons and evaluates the target season.

for target_season: validation_season = target_season - 1 for each stat s: choose n with best validation log loss for stat s fit final model using selected n per stat evaluate on target_season

The key leakage rule is consistent throughout: current-game stats and target-season outcomes are never used to create features or select windows for that same target season.

Finding Important Factors

Two importance checks are now included for the best global window. These are better first-pass tools than raw p-values because many basketball features are related to each other. For example, scoring margin, win rate, and points allowed are not independent.

  • Standardized coefficient: direction and size of a factor after putting features on comparable scales. Positive means the factor pushes the row's team toward a higher win probability.
  • Permutation log-loss delta: how much prediction quality gets worse when that feature is scrambled in the evaluation season. Bigger positive values mean the model relied on that feature more.

Current Takeaways

  • Yes, shorter windows were tested. They do not look competitive in the current backtest.
  • The strongest completed-season global window is currently n=20, narrowly ahead of n=10.
  • Adding basketball context improved the best completed-season global log loss from roughly 0.6260 to 0.6253.
  • The strongest factors in the first importance check are home court, recent margin difference, team recent margin, opponent recent margin, and points-allowed difference.
  • The next useful improvement is adding cleaner opponent-strength adjustment and running grouped ablation tests to compare whole feature families.
